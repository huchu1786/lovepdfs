FAQ

Common Questions

Quick answers before you reach out.

Are my files actually private? ▼
Yes — completely. All processing happens in your browser using WebAssembly. Your files never leave your device. We literally cannot see them even if we wanted to.
Why is PDF to Word an HTML/ZIP file? ▼
True .docx conversion requires server-side tools. Our browser-based version extracts text with layout analysis and packages it as HTML (which Word can open directly) + TXT. Open the HTML file in Microsoft Word, then File → Save As → .docx.
Is there a file size limit? ▼
There's no enforced limit — it depends on your device's RAM. Most modern browsers handle files up to 500MB comfortably. Very large files may slow down processing on older devices.
Why doesn't watermarking use full encryption? ▼
Full AES-256 encryption requires server-side processing because browsers don't expose the cryptographic primitives needed. Our protect tool adds metadata markers — for enterprise-grade encryption, use Adobe Acrobat.
Can I use iLovePDFs for commercial work? ▼
Yes! iLovePDFs is free for personal and commercial use. The only restriction is you must own or have permission to process the documents you upload.
Which browsers are supported? ▼
Any modern browser works: Chrome 80+, Firefox 75+, Safari 14+, Edge 80+. We recommend Chrome or Edge for the best WebAssembly performance.
